# Operator Surfaces

The AI4U Little Engineer platform has been extended with new operator-only views to monitor the health and performance of the intake, preview, and generation pipeline.

## The Daedalus Gate Inspector

The primary addition is the **Daedalus Gate Inspector** (`/admin/daedalus`). This dashboard provides a real-time, filterable view of all Daedalus Gate receipts generated by the system.

### Key Features

1. **Real-Time Monitoring**: Operators can see the last 50 receipts generated across all gates (e.g., `intake_interpretation`, `harmonia_merge`, `vpl`, `generation`).
2. **Filtering**: Receipts can be filtered by gate, result ("GO", "CLARIFY", "REJECT", "WARN"), session ID, and job ID.
3. **Metrics**: The dashboard displays key stats, including total receipts, counts for each result type, average confidence, and average latency.
4. **Payload Drill-Down**: Operators can expand any receipt to view the full JSON payload, including the merged request, VPL score breakdown, or Trust Policy Engine decision.

### Purpose

The Daedalus Gate Inspector is crucial for debugging and optimizing the platform. If a user's request fails or a generated part is rejected, operators can trace the exact sequence of decisions that led to that outcome, from the initial multimodal input to the final CAD generation step.

This level of transparency ensures the system remains reliable and accountable as it scales to handle more complex, multi-modal inputs.
